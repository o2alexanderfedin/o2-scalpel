**API audit (post Stage 1A T1+T13):**

- `LanguageServerProcess.add_notification_listener` (`src/solidlsp/ls_process.py:516`) is additive: multiple listeners coexist with the legacy primary handler, none clobber.
- `rust_analyzer.py:734` subscribes `_on_progress` via the same additive channel; `wait_for_indexing()` watches `rustAnalyzer/{Fetching,Building,Indexing,...}` + `rust-analyzer/flycheck/N` and signals when every observed indexing-class token reaches `end`.

**Decision:**

- A -> Stage 1A T1 ships `add_notification_listener` (additive subscriptions, no clobbering). T9/T13 wire `_on_progress` + `wait_for_indexing()` on top.
- B -> ship begin/end coverage; defer fine-grained to v0.2.0.
- C -> additive listener tees at the JSON-RPC dispatcher (alongside any primary).
