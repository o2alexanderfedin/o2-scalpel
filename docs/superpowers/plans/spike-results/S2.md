# S2 - snippetTextEdit:false honored by rust-analyzer assists (post Stage 1A)

**Outcome:** A - 0 of 45 resolved actions emit snippet markers; defensive strip is sufficient

**Evidence:**

- Code actions surfaced (10-range sweep over lib.rs): 45
- Resolved actions (after `codeAction/resolve`): 45
- Snippet-marker offenders (`$N` / `${N`): 0
- Capability advertised: `experimental.snippetTextEdit = false` (set by `RustAnalyzer.override_initialize_params`, T13)

**Offenders:**

_(none)_

**API audit (post Stage 1A T6/T7/T10/T13):**

- `RustAnalyzer.override_initialize_params` (rust_analyzer.py:683) flips `experimental.snippetTextEdit -> False` by default; the previous instance monkey-patch of `_get_initialize_params` is no longer needed and has been removed from this spike.
- `SolidLanguageServer.request_code_actions(file, start, end, only?, trigger_kind?, diagnostics?)` and `.resolve_code_action(action)` (T6/T7) replace the raw `srv.server.send_request(...)` calls.
- Deferred-resolution code actions (S3): every action MUST go through `codeAction/resolve` before `edit:` is materialized.
- Snippet regex `r"\$\d+|\$\{\d+"` covers `$0` placeholders + `${0:label}` / `${1|a,b|}` choices.

**Decision:**

- A -> defensive `$N` strip in applier (~10 LoC) suffices; no extra work.
- B -> mandatory regex strip path with placeholder/choice/escape edge-cases (+50 LoC).
