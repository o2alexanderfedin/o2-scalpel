## 1. Problem statement

`o2-scalpel` ships **35 task-level facades + 11 always-on primitive/safety/diagnostic tools** across 23 first-class language plugins (52 languages total) at velocity (52 langs in ~10 days). The engine in `vendor/serena/` carries **no coverage tooling**: no `pytest-cov`, no `.coveragerc`, no `[tool.coverage.*]` blocks, no CI lane. The existing 614-spike + 18-20 e2e regression suite is healthy but **invisible at the line/branch level** — we cannot answer "what fraction of the dispatch hot path runs under test?" or "what's the regression-risk surface of this PR diff?"

Bug history (from project memory) confirms the gap is real:

- **v0.2.0 review** surfaced 6 real bugs (facade arg-validation HIGH, fixture/coords mismatches, host pylsp gap) that line-coverage instrumentation would have made visible at PR time, not at audit time.
- **v1.6 STUB-facade fix:** 3 of 4 STUB facades shipped with passing unit tests but lied to the LLM ("dry_run_one_step lying-to-LLM"). Integration assertions on outcome-not-envelope would have caught the lie.
- **v1.7 rollback inverse-applier:** original rollback was a no-op disguised as a success. A property test asserting `apply(edit) ; rollback() == identity` round-trip would have caught it.

The user's stated profile (CLAUDE.md): "regression frustration: HIGH" — **flag potential regression risks explicitly**. Coverage instrumentation operationalizes that preference instead of leaving it to vibes.

## 2. Goals

1. **Visible coverage numbers** on the in-scope subset of `vendor/serena/src/`, published as a CI artifact on every PR.
2. **Bug-history-driven gap fill**: add integration tests + property tests + dead-code deletions targeting the seams where v0.2.0 / v1.6 / v1.7 bugs lived.
3. **Per-module structural gates** in CI on the load-bearing modules, only after gap-fill raises baseline numbers — never as the first step.
4. **Diff-coverage gate** on PR diffs (new code held to a higher bar than legacy debt) so velocity is not taxed by inherited under-coverage.
5. **No SaaS lock-in.** Coverage tooling stays in-tree (`pytest-cov`, `coverage.py`, `diff-cover`, `vulture`, `mutmut`, `hypothesis`) — no Codecov/CodeClimate dependency.
6. **Phase-wise rollback.** Each of the 3 phases (baseline, gap-fill, gates) is independently reversible.

## 3. Non-goals

- **Coverage on the 23 sibling `o2-scalpel-<lang>/` plugin trees individually.** They are generated by `o2-scalpel-newplugin`; testing the generator + one canonical instance (rust) covers all 23 (DRY).
- **Coverage on inherited Serena code o2-scalpel does not ship:** `vendor/serena/src/serena/jetbrains/`, `serena/generated/`, `serena/marketplace/*.py`, `serena/installer/_platform_*.py` adapters for platforms not in CI matrix, `interprompt/`.
- **Coverage on `scripts/`** — shell-thin; smoke-tested end-to-end via `uvx`.
- **A coverage badge in README.** Badges incentivize padding; the project-memory record explicitly warns against vanity-metric optimization.
- **SaaS coverage reporting** (Codecov / CodeClimate). Revisit at v2.5 if budget changes.
- **Whole-tree `--cov=serena`.** Always scoped to `serena.tools`, `serena.refactoring`, `serena.plugins`, `solidlsp`.
- **Performance / benchmark testing.** Separate concern, separate spec.
- **Replacing the existing routing-benchmark gate** at 53.3% top-1 (Phase-4 floor from `2026-04-29-lsp-feature-coverage-spec.md`). That stays as a behavior gate; this spec adds a structural gate alongside.

## 4. Decision rationale (adversarial brainstorm synthesis)

Three doctrines argued in parallel via the `superpowers:dispatching-parallel-agents` flow:

| Axis | Coverage-Maximalist | Test-Minimalist | YAGNI-Enforcer | Synthesis |
|---|---|---|---|---|
| Scope | `tools/` + `refactoring/` + `plugins/` + `solidlsp/` | same | same | **All three converged on this scope** |
| Exclude `generated/`, 22 sibling plugins, `marketplace/` JSON, `interprompt/`, `spikes/` | yes | yes | yes | **All converged** |
| Numeric target | 90% line / 80% branch, ratcheted | None — mutation score is real gate | Per-module floors, report-not-fail | **Per-module floors + diff-cover hard gate** (Phase C only) |
| Layer split | 70/25/5 unit/int/e2e | 20/50/30 (inverted pyramid) | e2e-heavy | **30/50/20** weighted toward integration |
| Special tools | `diff-cover` | `mutmut` + `hypothesis` | `vulture` | **All five adopted** at different phases |
| `O2_SCALPEL_RUN_E2E=1` env-gate | preserve | preserve (more valuable, not less) | preserve | **Preserved unchanged** |
| Existing 614-spike + e2e suite | augment | augment | the *real* gate | **Treated as floor; coverage adds visibility, not replacement** |

**Where each doctrine yielded ground:**

- **Maximalist → Minimalist** on test layer mix (50% integration vs. 25%) because the v1.6/v1.7 bug history shows behavior contradictions dominate, not uncovered-line bugs.
- **Minimalist → Maximalist** on unit-heavy testing for the dynamic capability registry specifically — pure logic with combinatorial state space where property tests + units genuinely beat integration.
- **YAGNI → Maximalist** on Phase C per-module gates (vs. YAGNI's "no numeric target") because the user's "regression frustration: HIGH" profile substitutes coverage gates for the second-reviewer line-by-line read that solo-dev workflow lacks.
- **All three** agreed solo-dev today justifies leaner stance; team growth would flip toward maximalist — **the spec captures this as a "revisit-on-team-growth" trigger, not a Phase D**.

The full position papers are preserved as adversarial brainstorm artifacts (agent output captured at brainstorm time; not committed to repo per YAGNI on intermediate artifacts).

## 5. Architecture & components

### 5.1 New dev dependencies (engine)

Added to `vendor/serena/pyproject.toml` `[project.optional-dependencies].dev` (or equivalent block):

| Dep | Version pin | Purpose | Phase introduced |
|---|---|---|---|
| `pytest-cov` | `>=5.0,<6.0` | Coverage instrumentation, parallel-safe via `coverage combine` | A |
| `vulture` | `>=2.10` | Dead-code detection on scoped modules; runs **before** coverage | A |
| `diff-cover` | `>=9.0` | PR-diff coverage gate (informational A; hard C) | A |
| `hypothesis` | `>=6.100` | Property tests on WorkspaceEdit applier + rollback inverse | B |
| `mutmut` | `>=3.0` | Nightly mutation testing scoped to `refactoring/` | B (dep added) / C (activated nightly) |

### 5.2 Coverage configuration (canonical)

In `vendor/serena/pyproject.toml`:

```toml
[tool.coverage.run]
source = [
  "serena.tools",
  "serena.refactoring",
  "serena.plugins",
  "solidlsp",
]
branch = true
parallel = true   # pytest-xdist compatibility
omit = [
  "*/serena/generated/*",
  "*/serena/jetbrains/*",
  "*/serena/marketplace/*.py",
  "*/serena/installer/_platform_*.py",
  "*/interprompt/*",
  "*/test/*",
  "*/spikes/*",
  "*/__pycache__/*",
]

[tool.coverage.report]
exclude_lines = [
  "pragma: no cover",
  "raise NotImplementedError",
  "if TYPE_CHECKING:",
  "if __name__ == .__main__.:",
]
show_missing = true
skip_covered = false
precision = 2
```

The 22 sibling `o2-scalpel-<lang>/` plugin trees live outside `vendor/serena/src/` and are auto-excluded by `source = […]`.

### 5.3 New CI workflow (`.github/workflows/coverage.yml`)

Three jobs:

1. **`fast-coverage`** — runs on every PR + push to `main`/`develop`. Phase A: artifact-only. Phase C: gates active.
2. **`e2e-coverage`** — nightly only. Matrix per language (`rust`, `python`, `markdown`, `typescript`, `go`, `cpp`, `java`, `csharp`). Each job pre-installs the host LSP binary, runs `O2_SCALPEL_RUN_E2E=1 pytest test/e2e/ --cov --cov-append`. Honest skip on missing binary (`continue-on-error: true`).
3. **`mutation`** — nightly only, Phase C+. `mutmut run --paths-to-mutate=vendor/serena/src/serena/refactoring/`. Informational artifact, no fail.

The existing Linux CI matrix (v1.3 milestone) and PyPI publish workflow are untouched — coverage is **additive**.

### 5.4 New helper scripts

- `scripts/coverage-floor-check.py` — Phase C+. Wraps `coverage report --include=… --fail-under=…` for each of the 4 in-scope modules. Exits non-zero on first floor breach.
- `scripts/check-mcp-tool-e2e-pairing.py` — Phase C+ drift gate. Greps `serena/tools/scalpel_facades.py` for `@tool`-decorated facade names and asserts each has at least one e2e test in `vendor/serena/test/e2e/` referencing it. Exits non-zero on missing pairing.

### 5.5 New test directory

- `vendor/serena/test/property/` — `hypothesis` property tests. Separate from `unit/` to keep fast-suite vs. property-run timing visible.

## 6. Phase plan

### Phase A — Baseline + visibility (no gate)

**Effort:** small (~1-2 days).

1. Add 3 dev deps (`pytest-cov`, `vulture`, `diff-cover`) to `vendor/serena/pyproject.toml`.
2. Add `[tool.coverage.run]` + `[tool.coverage.report]` blocks (§5.2).
3. Run `pytest --cov --cov-branch --cov-report=xml --cov-report=html --cov-report=term-missing` once on `main`. Capture per-module baseline numbers.
4. Run `vulture vendor/serena/src/serena/{tools,refactoring,plugins} vendor/serena/src/solidlsp` once. Catalog findings; **do NOT delete yet** (Phase B owns deletions).
5. Add `coverage.yml` CI workflow with `fast-coverage` job only. PR comment publishes coverage delta + vulture findings as artifact. **Does NOT fail the build.**
6. Add `docs/coverage-strategy.md` (parent repo) — living doc with baseline numbers + ratchet history. **No badge in README.**

**Exit criteria:** baseline numbers committed to `docs/coverage-strategy.md`; CI artifact appears on PRs; vulture findings catalogued.

**Rollback:** delete `coverage.yml`, drop dev deps, drop `[tool.coverage.*]` blocks. Repo state = pre-A.

### Phase B — Gap-driven critical-path tests

**Effort:** medium (~2 weeks). Add `hypothesis` + `mutmut` deps at start; `mutmut` activates in Phase C nightly.

Driven by the **bug-history hit list** (the load-bearing claim of the spec):

| # | Bug class | Module | New test type | Doctrine winning |
|---|---|---|---|---|
| B1 | v0.2.0 facade arg-validation (HIGH) | `serena.tools.scalpel_facades` | Integration: facade ↔ `MultiServerCoordinator.find_symbol_range` round-trip on real strategy registry | Min |
| B2 | v1.6 STUB lying-to-LLM | `serena.tools` (`apply_capability`, `split_python`, `dry_run_one_step`) | Integration: assert *outcome* (file changed on disk / disk unchanged) — not just envelope | Min |
| B3 | v1.7 rollback was a no-op | `serena.refactoring` (rollback inverse-applier) | `hypothesis` property test: `apply(edit) ; rollback() == identity` round-trip | Min |
| B4 | WorkspaceEdit applier idempotence | `serena.refactoring` | `hypothesis` property test: `apply(edit) ; apply(edit) == apply(edit)` | Min |
| B5 | DynamicCapabilityRegistry combinatorics | `serena.engine` | Unit + property: capability merge + dispatch gate | Max concession |
| B6 | Vulture-flagged dead code from Phase A | scoped modules | **Delete** (preferred) or `# vulture: keep <reason>` annotation with required reason text | YAGNI |
| B7 | Generator drift across 23 plugin trees | `o2-scalpel-newplugin` | Snapshot test: regenerate canonical (rust) plugin, diff against committed tree; CI fails on drift | Max compensating |

**Discipline rule (Min's gate):** every Phase B test starts with a `# regression: <bug-id>` comment linking back to the `.planning/` artifact or git tag where the bug was found. **If a Phase B test cannot be linked to a real historical bug, it does not get written.** This prevents Phase B from sliding into speculative coverage padding.

**Exit criteria:** every row B1-B7 has either a passing test or a deletion commit; vulture findings either deleted or annotated with required reason text; Phase A baseline numbers measurably moved up; new numbers captured in `docs/coverage-strategy.md`.

**Rollback:** new tests are pure-positive — leave them. Vulture deletions are commits — `git revert` if regretted.

### Phase C — Per-module gates

**Effort:** small (~3 days, mostly CI wiring).

Gates activate **only after Phase B raises numbers** (sequencing matters — gate-first creates a green-fail death-spiral).

**Per-module floors** enforced by `scripts/coverage-floor-check.py`:

| Module | Line floor | Branch floor |
|---|---|---|
| `serena.tools` | 80% | — |
| `serena.refactoring` | 85% | 70% |
| `serena.plugins` | 75% | — |
| `solidlsp` (core, excluding per-language adapters) | 70% | — |

**Diff-cover gate:** `diff-cover coverage.xml --compare-branch=main --fail-under=90` on the PR diff. New code held to 90% line; legacy debt does not block PRs (velocity preservation — Maximalist's own concession).

**Drift gate:** `scripts/check-mcp-tool-e2e-pairing.py` fails CI on new top-level MCP tool added without paired e2e (YAGNI's win).

**Vulture gate:** new vulture-flagged dead code without `# vulture: keep <reason>` annotation fails CI.

**Nightly jobs:**

- `e2e-coverage` matrix — host-binary-pre-installed e2e per language with `--cov-append`. Combined coverage report uploaded.
- `mutation` — `mutmut run --paths-to-mutate=vendor/serena/src/serena/refactoring/`. Mutation score artifact published. **Below 60% surfaces a notification but does NOT fail** — Min's mutation-as-quality-signal, not as hard gate.

**Exit criteria:** all four floors green in CI; diff-cover gate active on PRs; nightly mutmut publishes a number weekly without failing the build; drift gate live; vulture gate live.

**Rollback:** remove `--fail-under` and per-module floor checks from `coverage.yml`. Repo falls back to Phase A (informational).

## 7. Test layer allocation

Target ratio (by test count) across the engine: **~30% unit / ~50% integration / ~20% e2e**, weighted by effort toward integration.

| Layer | Home | Bias | Notes |
|---|---|---|---|
| Unit | `vendor/serena/test/unit/` | Pure functions: capability resolution algebra, edit-coalescing, path normalization | If it needs a mock LSP, promote to integration |
| Integration | `vendor/serena/test/integration/` + `vendor/serena/test/serena/` | Cross-module: facade → strategy → coordinator → applier; no real LSP processes | This is where v1.6/v1.7 bugs would have been caught |
| E2E | `vendor/serena/test/e2e/` (gated by `O2_SCALPEL_RUN_E2E=1`) | One happy-path per facade × language; honest skips on missing host binary | Honest skips are a feature, not a bug — locally skip-when-absent, CI matrix installs binary per language |
| Per-language LSP | `vendor/serena/test/solidlsp/` | One round-trip handshake per shipped adapter | Stop there — deeper adapter tests duplicate the LSP server's own suite |
| Property | `vendor/serena/test/property/` (new) | `hypothesis` invariants on WorkspaceEdit applier + rollback inverse + capability merge | Pinned profile `settings(deadline=None, derandomize=True)` for CI; full-random nightly |
| Spikes | `vendor/serena/test/spikes/` | Frozen — no new tests, not gated, not measured | Research yard preserved |

## 8. Error handling & edge cases

| Scenario | Behavior |
|---|---|
| `pytest-xdist` worker crashes mid-run | `coverage combine` ignores incomplete `.coverage.*` files; report missing-worker warning |
| Coverage drops on legacy untouched code | Per-module floor catches it; PR fails with module name + delta (Phase C only) |
| New file lands without coverage | `diff-cover --fail-under=90` catches it on PR diff (Phase C) |
| New MCP tool without e2e | Drift-gate script fails CI (Phase C) |
| Host LSP binary missing in nightly matrix | Matrix job for that lang `continue-on-error: true`; reports as honest skip |
| `mutmut` hits an equivalent mutant | Triage budget: ~30 min/quarter manually annotated as `# mutmut: equivalent` — non-failing |
| `hypothesis` finds a real bug | Test fails, pytest output shows shrunk counter-example, reproducer auto-saved by hypothesis to `.hypothesis/` |
| Vulture false-positive | `# vulture: keep <reason>` inline annotation; lint rule requires the reason text be non-empty |
| `coverage combine` fails on parallel artifacts | CI step retries once; on second failure, surface raw `.coverage.*` files as artifact for offline triage |

## 9. Risks & mitigations

| Risk | Likelihood | Mitigation |
|---|---|---|
| Velocity tax (Maximalist's own concession) | Med | Phase A is zero-gate; Phase B is gap-fill (writing the test you'd write anyway); Phase C only gates *changed code* via diff-cover so legacy debt doesn't block PRs |
| Tests-of-mocks rot (mocking the LSP boundary lies) | Med-High | Bias allocation toward integration/e2e (50%/20%); `hypothesis` property tests on `WorkspaceEdit` test invariants, not behavior-shapes; `solidlsp/` transcript layer for protocol determinism |
| `mutmut` slow + noisy | High | Nightly only, scoped to `refactoring/` (~tightly bounded), informational not gating, equivalent-mutant triage budget |
| Generator drift (only 1 of 23 plugin trees tested) | Low-Med | Snapshot test (B7) regenerates rust plugin and diffs; CI fails on drift |
| Solo-dev coverage-as-substitute-reviewer fragility | Always-on | Phase C diff-cover at 90% specifically substitutes for the second-reviewer line-by-line read |
| Phase C gates trip on inherited Serena code we haven't touched | Med | Floors scoped to the 4 modules we own (`serena.tools`, `.refactoring`, `.plugins`, `solidlsp` core); inherited `jetbrains/`, `marketplace/`, `generated/` excluded |
| `hypothesis` flake from random seeds | Low | CI profile pinned via `settings(deadline=None, derandomize=True)`; full-random for nightly |
| Coverage instrumentation slows pytest > 30% | Low-Med | Measured during Phase A; if exceeded, narrow `source = […]` further or run `--cov` only on coverage CI lane (not on every developer pytest invocation) |

## 10. Self-watching ("who watches the watchman" — TRIZ)

- `coverage.yml` workflow tested via `act` (local GitHub Actions runner) before merge.
- `coverage-floor-check.py` is itself a Python script with its own unit tests in `test/unit/`.
- `check-mcp-tool-e2e-pairing.py` is itself a Python script with its own unit tests.
- The bug-history hit list in §6 Phase B is verified by **regenerating each historical bug as a regression test** — every test starts with `# regression: <bug-id>` linking back to the `.planning/` artifact or git tag.

## 11. Revisit triggers

The spec assumes solo-dev today. Re-open this spec if:

- **Team grows past solo.** Shared codebase needs legible structural signal; mutation score is too expert-only to be the primary metric.
- **A regression escapes from an excluded zone** (`jetbrains/`, `_StubStrategy` dead branches, generated trees). One real escape justifies pulling that zone back into scope.
- **Mutation testing produces < 10% actionable signal after one quarter.** Drop `mutmut`; keep integration-heavy testing without mutation overhead.
- **Phase C gates trip on legitimate refactor work > 1×/quarter.** Lower floors or rely on diff-cover only.
- **A coverage-padding incident** is observed (test added that touches lines without asserting outcomes). Add mutation-score-on-changed-files to PR gate.
