## Post body

**Just shipped v1.14 of o2.scalpel — 52-language refactoring for AI coding agents, packaged as Claude Code plugins.**

→ **https://github.com/o2alexanderfedin/o2-scalpel**

The foundation is the excellent **Serena MCP server** by Oraios AI — it already wraps LSP write operations (`rename_symbol`, `replace_symbol_body`, `safe_delete`, `applyEdit`) as MCP tools. Real LSP calls routed through `LanguageServerCodeEditor`, not regex. That's the hard part, and it was already solved.

What o2.scalpel adds on top of that foundation:

→ A **task-level facade layer** — 35 `scalpel_*` tools (`split_file`, `extract`, `inline`, `change_visibility`, `complete_match_arms`, `extract_lifetime`, `convert_to_async`, `rename_heading`, `generate_constructor`, etc.) that compose Serena's primitives + `textDocument/codeAction` into refactor-shaped operations an LLM can call by name.

→ **Dynamic LSP capability gating** — Pyright's missing `textDocument/implementation` returns `CAPABILITY_NOT_AVAILABLE` at dispatch instead of a slow `SYMBOL_NOT_FOUND`.

→ **52-language Claude Code marketplace** — 23 with curated facade sets (Rust, Python, TypeScript, Go, Java, Markdown, …) and 29 generated as minimal plugins from a universal `rename_symbol` + `fix_lints` pair covering the LSP long tail (Bash, Lua, Swift, Kotlin, Scala, Vue, Solidity, Terraform, Zig, Julia, …).

```
/plugin marketplace add o2alexanderfedin/o2-scalpel
/plugin install o2-scalpel-rust@o2-scalpel
```

Spun out of an internal C++→Rust transpiler project on April 24th. Nine days from spinout to 52-language marketplace, built collaboratively with the agent. The project is the proof: when you let the agent design with you instead of just type for you, the velocity equation genuinely changes.

Massive thanks to **Oraios AI** for Serena — Scalpel is meaningless without it.

→ **https://github.com/o2alexanderfedin/o2-scalpel**
